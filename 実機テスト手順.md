# onsa 実機プライベートテスト手順

> 2026-04-17 作成 / 対象: Even Realities G2 (実機)
> 前提: シミュレーターで既に動作確認済み（別途やる場合は `実行手順.md` を参照）

ストア公開せずに開発PCの dev サーバーを iPhone Even Hub アプリから読み込む、最短プライベート検証の流れ。

---

## 0. 事前準備

| 項目 | 内容 |
|------|------|
| iPhone | Even Hub アプリがインストール済み・G2 とペアリング済み |
| Wi-Fi | 開発PCと iPhone を **同一ネットワーク** に接続 |
| PCファイアウォール | ポート 5173 をローカルLANに開放（初回接続時に Windows Defender が聞いてくる） |
| PC IP | `ipconfig` で LAN IP を確認（例 `192.168.1.42`） |

**重要**: `getUserMedia`（内蔵マイク）は iOS WebView で **HTTPS または localhost でのみ** 動作する。LAN経由なら HTTPS 必須。本アプリでは `@vitejs/plugin-basic-ssl` で自己署名証明書を出す。

---

## 1. HTTPS dev サーバーを起動

```bash
cd "D:/Download/Claude PJ/EVEN HUB/onsa"
npm run dev:https
```

出力例:
```
  VITE v7.3.2  ready in 240 ms
  ➜  Local:   https://localhost:5173/
  ➜  Network: https://192.168.1.42:5173/
```

**自己署名証明書の初回受け入れ**:
1. iPhone の **Safari** で `https://192.168.1.42:5173/` を開く
2. 「接続はプライベートではありません」警告 →「詳細を表示」→「このWebサイトを訪問」
3. Safari で onsa の画面が表示されれば成功 → マイク許可ダイアログが出たら許可し、ノート表示が動くことを確認
4. ここで動かない場合は Even Hub WebView でも動かないので、このレイヤーで解決すべし

## 2. QR を生成して Even Hub アプリから読み込ませる

別ターミナルで:

```bash
cd "D:/Download/Claude PJ/EVEN HUB/onsa"
npm run qr:https
```

QR が端末に表示される（URL: `https://<PCのIP>:5173/`）。

iPhone の **Even Hub アプリ**:
1. 右上（または設定）の **QR スキャン** メニューを選択
2. PC画面の QR を読み取る
3. Even Hub 内 WebView で onsa が開く
4. **Connect Glasses** をタップ → G2 にチューナー画面が出れば成功
5. **Toggle Mic** を押してマイク許可ダイアログが出たら許可

## 3. 期待する動作

- G2 グラスのディスプレイに3行表示:
  ```
  A#4      466.23 Hz
  |---------+----*----|  +12c
  Target 466.16Hz  RMS 0.12
  ```
- 音を出すと Hz と音名とメーターがリアルタイムに更新される
- 音を止めると `-- listening...` に戻る
- ±5 cent 以内で（ブラウザ側のみ）針が緑化

## よくあるトラブル

### A. Even Hub WebView でマイク許可ダイアログが出ない / 録音できない

iPhone の Even Hub アプリの WebView がマイク権限を要求しない仕様だった場合、`getUserMedia` は失敗する。回避策:

1. **iOS 設定 → Even Hub → マイク** で権限を事前ONにする
2. それでもダメなら iPhone 側のアプリ実装の制約なので、一旦 **Safari で直接 URL を開いて** テストする（G2への送信はされないがピッチ検出ロジックの確認は可能）
3. 将来的には iPhone に音声を流すのではなく、**G2 本体マイク → BLE → Bridge 経由** で取得する API が SDK に無いか確認（現行 SDK v0.0.7 には音声入力 API が見当たらない）

### B. HTTPS 証明書エラーで開けない

- Safari で一度開いて証明書を受け入れてから Even Hub WebView で開く
- それでもダメな場合: [mkcert](https://github.com/FiloSottile/mkcert) で信頼済み証明書を発行し、`vite.config.ts` で `https: { key, cert }` を直指定する

### C. `Network` の IP が 0.0.0.0 になる

PC の Wi-Fi が切れているか VPN 経由。`ipconfig` で実 IP を確認して `--ip` フラグを使う:

```bash
npx evenhub qr --https --ip 192.168.1.42 --port 5173
```

### D. G2 への表示が真っ暗 / 反応しない

- Even Hub アプリ内で G2 が "Connected" になっているか確認
- ブラウザコンソールで `bridge ready, page rendered` ログが出ているか確認（Safari のリモートデバッガでPCから覗く）
- `waitForEvenAppBridge` が timeout(4s) した場合は "browser-only mode" になるので、Connect をもう一度押す

### E. 低音（E2=82Hz等）が取れない

`lib/pitch.ts` の `MIN_FREQ` を 40 に、FFT_SIZE (main.ts) を 4096 に上げる（ただし遅延は増える）。

## 実機テスト完了の判定チェックリスト

- [ ] PC の Safari で `https://localhost:5173/` でマイク動作確認
- [ ] iPhone の Safari で `https://<LAN-IP>:5173/` でマイク動作確認（証明書受け入れ済）
- [ ] Even Hub アプリから QR スキャンで onsa が開く
- [ ] Connect Glasses で G2 に初期画面（-- listening...）が出る
- [ ] マイクに音を出すと G2 の3行がリアルタイムに更新される
- [ ] ギター/ピアノなど複数楽器で音名が妥当（A=440Hz 基準）
- [ ] Toggle Mic でマイクのON/OFFが切り替わる

ここまで通れば実機プライベートテスト完了。
