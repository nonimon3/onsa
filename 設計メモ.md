# onsa — Even G2 チューナーアプリ 設計メモ

> 作成日: 2026-04-17
> 対象: Even Realities G2 (Even Hub / even-dev)
> アプリID: `onsa`

---

## 1. 目的

内蔵マイクから音声を取り込み、ピッチ（基本周波数）を解析して以下をG2グラスとブラウザ両方に表示するリアルタイム・チューナー。

- 最も近い半音（音名）: 例 `A#4`
- 実測周波数: 例 `466.23 Hz`
- ズレ（セント値）: -50 ~ +50 cent
- 視覚的メーター（Unicode バー）でズレを表示

## 2. スコープ

- 内蔵マイク (`navigator.mediaDevices.getUserMedia`) から単音（モノフォニック）を解析
- 楽器チューニング、ボーカル、口笛など幅広く対応
- 検出レンジ: 約 50 Hz ~ 2000 Hz（ギター/ベース/ボーカル含む）
- しきい値以下（無音）は `--` 表示
- ノート基準: 標準 A4=440Hz （後で設定可能にする余地あり）

## 3. アーキテクチャ

```
[内蔵マイク]
    │ getUserMedia
    ▼
[AudioContext + AnalyserNode (time-domain)]
    │ Float32Array (2048 sample)
    ▼
[autoCorrelate() ピッチ検出]
    │ 周波数 Hz
    ▼
[frequencyToNote() 音名変換]
    │ { note, octave, cents, targetHz }
    ├──▶ ブラウザUI更新
    └──▶ G2 コンテナ更新（textContainerUpgrade）
```

## 4. ピッチ検出アルゴリズム

### 採用: 自己相関 (ACF) + パラボリック補間

理由:
- 実装コスト低い (~50行)
- モノフォニック楽器/音声で高精度
- FFT より低遅延（時間領域）
- YIN の簡易版として実績あり

処理手順:
1. AnalyserNode で `getFloatTimeDomainData()` → Float32Array(2048)
2. RMS で音量判定 → しきい値 `< 0.01` は無音扱い
3. DCオフセット除去（平均値を引く）
4. 自己相関: `r[τ] = Σ x[i]*x[i+τ]`
5. 最初のピークを探索（短すぎる τ は除外）
6. 放物線補間でサンプル単位未満の精度を得る
7. `freq = sampleRate / bestOffset`

### 音名変換

```
noteNumber = 69 + 12 * log2(freq / 440)   // MIDI note number (float)
rounded    = round(noteNumber)
cents      = (noteNumber - rounded) * 100
targetHz   = 440 * 2^((rounded - 69) / 12)
noteName   = ['C','C#','D','D#','E','F','F#','G','G#','A','A#','B'][rounded % 12]
octave     = floor(rounded / 12) - 1
```

## 5. ディスプレイ設計（G2: 576 x 288 px, 緑グレースケール）

```
┌──────────────────────────────────────────────────┐
│  A#4              466.23 Hz                      │  ← 行1 (大きめ)
│                                                   │
│  ├─────────────●─────────────┤    -12c           │  ← 行2 メーター
│                                                   │
│  Target 466.16 Hz   RMS 0.12                     │  ← 行3 詳細
└──────────────────────────────────────────────────┘
```

- メーターは21セル (中央=0c、1セル=5c、両端=±50c)
  記号: `├` `┤` 端、`─` セル、`●` 現在位置
- `isEventCapture: 1` のテキストコンテナを1つ含める（操作要件のため）
- `textContainerUpgrade` で文字部分を毎フレーム（150ms 程度）更新

## 7. even-dev への取り込み

方式A（推奨・開発中）: `apps.json` にローカルパスを追加

```json
{
  ...,
  "onsa": "../onsa"
}
```

`./start-even.sh onsa` で起動可能になる。

方式B: `even-dev/apps/onsa/` に直接配置してビルトインとして登録。
ただし AppModule 規約（`createActions(setStatus)` を返す）に従う必要あり。

本プロジェクトは **方式A を基本**とし、同時に `index.ts` で AppModule を export して方式Bでも動くようにする。
