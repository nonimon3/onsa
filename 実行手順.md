# onsa 実行手順

## A. 単体ブラウザテスト（最初はこっちで動作確認）

```bash
cd "D:/Download/Claude PJ/EVEN HUB/onsa"
npm install
npm run dev
```

1. ブラウザで http://localhost:5173 を開く
2. "Connect Glasses" をクリック → G2 ブリッジは見つからないので "browser-only mode" になる
3. 自動でマイクの許可ダイアログが出る → 許可
4. 口笛や声、ギターなどで音を出すと、画面上部の大きな音名と Hz、セントメーターが動く
5. ズレ ±5 cent 以内で針が緑になる

> この段階で動けば、ピッチ検出ロジックは健全。

## B. even-dev シミュレーターで G2 画面を確認

### B-1. apps.json にローカルパスを追加

`D:/Download/Claude PJ/EVEN HUB/even-dev/apps.json` を編集:

```json
{
  "chess": "https://github.com/dmyster145/EvenChess",
  ...
  "smart-cart": "https://github.com/bryan-datastorm/smart-cart-even-g2",
  "onsa": "../onsa"
}
```

### B-2. シミュレーター起動

```bash
cd "D:/Download/Claude PJ/EVEN HUB/even-dev"
./start-even.sh onsa
```

（Windows では Git Bash / WSL から）

あるいは:

```bash
APP_PATH=../onsa APP_NAME=onsa ./start-even.sh
```

音声デバイスを明示したい場合:

```bash
# 利用可能デバイス一覧
npx @evenrealities/evenhub-simulator@latest -b default --list-audio-input-devices

# 例: Windows の内蔵マイク
AUDIO_DEVICE="coreaudio:BuiltInMicrophoneDevice" ./start-even.sh onsa
```

### B-3. 期待する動作

- ブラウザが立ち上がる（http://localhost:5173）
- シミュレーター（緑のグラス画面）に以下の3行が表示される:
  ```
  A#4      466.23 Hz
  |---------+----*----|  +12c
  Target 466.16Hz  RMS 0.12
  ```
- 音を止めると "-- listening..." に戻る

## トラブルシュート

| 症状 | 原因と対処 |
|------|------------|
| マイク許可ダイアログが出ない | ブラウザのサイト設定で localhost のマイクをリセット |
| 音名が暴れる | 背景騒音が大きい → 静かな環境で試す。`SMOOTH_WINDOW` を増やす |
| 低い音（E2=82Hz 等）が検出されない | `lib/pitch.ts` の `MIN_FREQ` を 40 等に下げる |
| G2 の文字が欠ける | `ROW*_WIDTH` を短くして width を下げる |
| textContainerUpgrade が効かない | ブラウザコンソールのエラー確認。必要なら `rebuildPageContainer` にフォールバック追加 |
